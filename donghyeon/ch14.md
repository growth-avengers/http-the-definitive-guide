# ch14. 보안 HTTP

- HTTPS
  - 모든 HTTP 요청, 응답 데이터는 네트워크로 보내지기 전에 암호화
  - HTTP(애플리케이션 계층)와 TCP(전송 계층) 사이에 '전송 레벨 암호 보안 계층'을 제공
    - 안전 소켓 계층(Secure Sockets Layer, SSL)
    - 전송 계층 보안(Transport Layer Security, TLS)
  - HTTP 프로토콜에 대칭, 비대칭 인증서 기반 암호 기법을 결합
  - 데이터 전송 과정
    1. 443 포트로 TCP 커넥션
    2. SSL 보안 매개변수 핸드셰이크를 통해 SSL 계층 초기화
    3. SSL 닫힘
    4. TCP 커넥션 닫힘
  - SSL 핸드셰이크
    - SSL 핸드셰이크가 하는 일
      - 프로토콜 버전 번호 교환
      - 양쪽이 알고 있는 암호 선택
      - 양쪽의 신원을 인증
      - 채널을 암호화하기 위한 임시 세션 키 생성
